# Robot-Based UWB Localization Testbed — Processing Pipeline

This repository contains the data processing pipeline developed for a Robot-Based UWB Localization Testbed and Simulation Environment, completed as part of an M.Sc. Embedded Systems thesis at TU Chemnitz (2025).

The pipeline processes ROS 2 bag files recorded during localization experiments, validates UWB positioning data against wheel odometry, and produces spatial accuracy metrics and visualizations.

---

## Pipeline Overview

The pipeline consists of 8 stages, each implemented as a standalone Python script:

| Script | Stage | Description |
|--------|-------|-------------|
| `1_extract_all_topics.py` | Extract | Reads a ROS 2 bag file and exports UWB pose and odometry data to CSV |
| `2_remove_zero_xyz_rows.py` | Clean | Removes all-zero rows caused by a data collection bug in early recordings |
| `3_sync_and_shift_all_to_uwb.py` | Sync | Shifts UWB coordinates to origin and synchronizes odometry to UWB timestamps |
| `4_trajectory_all_high.py` | Trajectory | Generates meter-wise trajectory JSON for the high-mounted UWB configuration |
| `4_trajectory_all_low.py` | Trajectory | Generates meter-wise trajectory JSON for the low-mounted UWB configuration |
| `5_voxel_hits_batch_converter.py` | Convert | Converts OSM3DM voxel hit JSON files to a normalized coordinate frame |
| `6_kdtree.py` | Error | Uses KDTree spatial matching to compute 2D localization error per timestamp |
| `7a_plot_k_Manual.py` | Visualize | Plots voxel value vs error using manual bin width |
| `7b_plot_conventionals.py` | Visualize | Plots voxel value vs error using Sturges, sqrt, and Freedman-Diaconis binning |
| `7c_plot_sturges.py` | Visualize | Plots voxel value vs error using Sturges binning with raw scatter overlay |
| `7d_plot_regressions.py` | Visualize | Plots voxel value vs error with linear, quadratic, and exponential regression fits |
| `8_compute_position_accuracy_batch.py` | Metrics | Computes and exports standard 2D positioning accuracy metrics (RMSE, CEP50, R95, etc.) |

## System Requirements

### Script 1 only
- Ubuntu (tested on Ubuntu 22.04)
- ROS 2 (tested on ROS 2 Humble)
- Python packages: `rclpy`, `rosbag2_py`, `rosidl_runtime_py`

### Scripts 2–8
- OS-independent (Windows, Linux, macOS)
- Python 3.8+
- Required packages:
  ```
  pip install pandas numpy scipy matplotlib scikit-learn
  ```
  Note: `scikit-learn` is only required for `7d_plot_regressions.py`.

## Usage

Each script is designed to be run from within the folder containing the relevant data files. Place the script in the same directory as its input files and run:

```bash
python <script_name>.py
```

> **Important:** Each script expects to find its input files in the same folder as the script itself. Do not run scripts from a different working directory.

### Stage 1 — Bag Extraction
Place `1_extract_all_topics.py` in a folder containing **exactly one** `.db3` ROS 2 bag file. The script will automatically locate and process it.

### Stages 2–8
Run each script in sequence. Outputs from each stage are consumed as inputs by the next.

## Frame Conventions and Coordinate Notes

### Odometry frame correction (Script 1)
The odometry topic publishes in the robot's local frame. X and Y signs are negated to align with the UWB global frame convention used in this testbed.

### UWB scaling (Script 1)
Raw UWB pose coordinates are published in decimeters. Script 1 divides all UWB coordinates by 10 to convert to meters, matching the odometry frame units.

### Axis remapping (Scripts 4 and 5)
Coordinates are remapped between the CSV frame and the world frame used by OSM3DM:
- CSV `x` → World `x`
- CSV `z` → World `y`
- CSV `y` (negated) → World `z`

### UWB mounting configurations (Scripts 4a and 4b)
Two UWB mounting heights were tested during the experiments:
- **High mount:** `OFFSET = {"x": 47.5, "y": 1.8, "z": -2.2}`
- **Low mount:** `OFFSET = {"x": 47.5, "y": 1.14, "z": -2.2}`

These offsets align UWB coordinates with the OSM3DM world frame origin.

### Timestamp synchronization (Script 3)
Odometry is synchronized to UWB timestamps using nearest-neighbor matching with a **0.5-second tolerance window**. Odometry readings more than 0.5 seconds from a UWB timestamp are discarded.

---

## OSM3DM Dependency (Script 5)

Script 5 requires voxel hit JSON files generated by **OSM3DM**, an internal simulation tool that is not publicly available. These files must be manually copied into the script's folder before running.

The converted output files are not included in this repository. As a result, scripts 6–8 also cannot be run without OSM3DM access, since they depend on Script 5's output.

The source code for all scripts is provided for reference and reproducibility of methodology.

---

## Pipeline Limitations

- The pipeline was developed and tested for a specific lab environment. Coordinate offsets, scaling factors, and frame conventions are specific to this testbed setup.
- AMCL localization was explored during development but ultimately not used. All comparisons are between UWB positioning and wheel odometry only.
- Script 2 addresses a data collection bug present in early recordings. It is not required for the published bag files but is retained for completeness.
